# AI 摘要优化方案

## 目标

围绕 `lib/plugins/aiSummary.js` 的 AI 摘要链路，优化以下四个环节：

1. 正文预处理
2. 提示词生成
3. 响应提取
4. 摘要清洗

目标是提升技术文章摘要质量、减少误删关键信息、增强不同 AI 服务响应格式兼容性，并让长文摘要更稳定。

---

## 现状问题

### 1. 正文预处理
- 当前对特殊字符过滤过严，容易破坏 `Next.js`、`Node.js`、`C++`、`A/B Test` 这类技术词。
- 行内代码、链接文本、部分公式说明会被整体删除，导致信息损失。
- 截断策略是简单 `slice(0, wordLimit)`，长文后半段重点无法进入摘要输入。

### 2. 提示词
- 当前更偏“格式约束”，缺少“摘要内容优先级”的明确指引。
- 没有根据正文长度动态调整摘要长度目标。
- 缺少对“避免模板句、空话、泛化表述”的约束。

### 3. 响应提取
- 目前兼容 `choices[0].message.content`、`content`、`result`、`text`，但未兼容 `message.content` 为数组的情况。
- 若未来接入 SSE 风格或非标准 JSON，容错不足。
- 对空摘要、超短摘要没有单独校验。

### 4. 摘要清洗
- 当前会全局移除较多字符，可能误伤技术表达。
- 前缀清理覆盖不够全面。
- 缺少摘要长度、句末标点、重复片段等后处理。

---

## 改进方案

### 一、正文预处理改进

#### 方案
1. 将“激进删除”改为“保守去噪”：
   - 保留技术文中常见符号：`.` `-` `/` `:` `%` `+` `#`
   - Markdown 链接保留文本，删除 URL
   - 行内代码保留内容本身，不直接整体清空
2. 对代码块、图片、HTML 等无关内容继续移除。
3. 增加段落化采样截断策略：
   - 保留标题
   - 保留 TOC 前若干项
   - 正文优先采样：开头 + 中间 + 结尾
4. 保持输入结构：标题、文章结构、文章内容。

#### 预期收益
- 技术词保真度更高
- 长文摘要覆盖更全面
- 降低因预处理过猛导致的摘要空泛问题

---

### 二、提示词改进

#### 方案
1. 在系统提示中加入“摘要内容优先级”：
   - 主题
   - 关键观点 / 方法 / 步骤
   - 结论 / 结果 / 价值
2. 增加文体兼容说明：
   - 教程文：提炼目标、关键步骤、适用对象
   - 技术文：提炼问题、方案、结果
   - 观点文：提炼核心论点与结论
3. 按正文长度动态设定摘要目标字数。
4. 明确禁止模板句：
   - 如“本文介绍了”“文章主要讲了”等

#### 预期收益
- 摘要更像信息提炼，而不是表面复述
- 减少空话、套话
- 长短文摘要更协调

---

### 三、响应提取改进

#### 方案
1. 抽出统一的响应提取函数。
2. 兼容以下返回形式：
   - `choices[0].message.content` 为字符串
   - `choices[0].message.content` 为数组
   - `choices[0].text`
   - `content`
   - `result`
   - `text`
3. 在提取后统一判空和最小长度校验。
4. 保留更清晰的错误日志，便于排查服务商格式差异。

#### 预期收益
- 增强多模型 / 多服务兼容性
- 降低“接口成功但提取失败”的概率

---

### 四、摘要清洗改进

#### 方案
1. 继续移除思考标签、Markdown 包裹、明显前缀。
2. 避免全局删除普通技术字符。
3. 增加后处理：
   - 合并重复标点
   - 删除模板开头
   - 自动补句号
   - 超短摘要判定为无效
4. 保持“纯文本单段输出”。

#### 预期收益
- 技术文摘要更自然
- 输出更稳定
- 降低误伤正常表达的风险

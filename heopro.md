## **💡 当 Heo 遇上 macOS**

Heo 是一款充满活力且相当完美精致的主题，所以我想在此基础上做些锦上添花的设计，我的想法是一种更克制、更具“材质感”的表达方式。我希望我的博客不仅是一个信息载体，更像是一台精致的 MacBook 屏幕，让阅读变成一种享受。

于是，我对现有的 Heo 主题进行了一次深度的“基因重塑”。

## **🛠 设计语言的重构**

### **1. 全局材质：从色块到玻璃**

我们彻底放弃了原生的硬色块，引入了一套系统级的**Design Tokens**。

- **底色**：使用了 macOS 系统同款的中性灰（Light #f5f5f7 / Dark #0f1115）。
- **玻璃拟态**：将导航栏与卡片容器全面升级为半透明材质，配合 backdrop-filter: saturate(180%) blur(16px)，营造出 Safari 般的通透感。
- **细描边与光影**：用 1px 的极细描边替代沉重的阴影，让层级更加细腻。

### **2. 阅读体验：排版调整**

为了解决长文阅读的疲劳感，我将排版系统调整些许：

- **呼吸感**：将正文行高锁定在 1.78，配合 0.9em 的段间距。
- **画框感图片**：文章内的图片被赋予了 12px 的圆角与微弱阴影，让每一张图都像是一件艺术品。
- **极简引用**：重塑了 blockquote，去掉了浮夸的装饰，仅保留纯净的排版节奏。

### **3. 代码块：仿造Mac风格**

代码是程序员的语言，它的质感至关重要。

- **Mac 窗口风格**：保留经典的三色点，并将代码块在浅色模式下也默认为暗色背景，增加视觉冲击力。

```c
#include <stdio.h>
printf("如你所见，这是一行代码")
```

### 4. 更简洁大气的footer

**移动端**的Footer bottom 会是三段纵向排列：

- left（PoweredBy/作者）
- center（社交图标）
- right（备案/统计）

**桌面端**的会变成横向并列，社交图标在中间且不会撑高 Footer。

### 5. 导航栏与交互：发光胶囊 + 有色磨砂玻璃

- 顶部导航菜单项在 **hover / focus / active** 状态下呈现“胶囊”形态。
- **亮色模式**使用蓝色作为高亮基调，**暗色模式**使用黄色作为高亮基调。
- 下拉菜单升级为“大圆角胶囊容器”，并为菜单项提供一致的有色高亮反馈。
- 通过 `backdrop-filter` 与半透明有色背景（tinted glass），让高亮不再是实色块，而是带材质感的磨砂玻璃。

### 6. 背景氛围：轻纹理增强磨砂表现

`backdrop-filter` 的磨砂效果依赖“背后有内容可被模糊”。为此我们在亮色/暗色背景上引入了极其克制的微纹理：

- **亮色模式**：在 `#f7f9fe` 底色上加入极淡的蓝色点状纹理。
- **暗色模式**：加入更淡的白色点状纹理。

这使得导航栏、下拉菜单、footer、分类栏等玻璃材质在视觉上更通透、更有层次。

### 7. 首页分类栏（CategoryBar）：磨砂玻璃与胶囊分类项

首页分类栏作为视觉焦点之一，同步升级为 macOS 风格的玻璃组件：

- 分类栏容器采用半透明磨砂背景 + 大圆角。
- 分类项升级为胶囊按钮，hover/selected 时提供与导航一致的蓝/黄有色高亮与光晕。
- 容器 hover 时增加轻微外圈光晕，强化层级。

### 8. 智能环境色：基于封面的氛围感

为了打破单一色调的沉闷感，我们引入了**智能环境取色系统**。

- **文章头图感应**：文章页顶部背景不再是固定的蓝色，而是自动从文章封面图中提取主色。
- **深度处理**：提取出的色彩会经过 62% 的压暗处理（Darken Logic），确保文字可读性的同时，营造出一种电影感的沉浸背景。
- **侧边栏同步**：侧边栏的个人资料卡（InfoCard）在文章页内也会同步该环境色，增强整体视觉的包裹感。

### 9. 细节微调：更高维度的视觉对齐

- **通透底色**：将全局背景从之前的 `#f7f9fe` 提升至明度更高的 `#f9fafc`，消除“灰蒙蒙”的感觉，让页面更显高级、通透。
- **Footer 无缝融合**：重构了底部的过渡逻辑，采用 `transparent -> body` 底色的渐变，并配合 80% 不透明度的玻璃材质，让 Footer 与页面背景自然衔接，消除视觉断层。
- **组件透传增强**：对底层 `Card` 组件进行了属性透传重构，支持实时注入 CSS 变量，为未来的动态样式扩展打下了坚实基础。

### **10. 灵动岛播放器：有生命感的音乐交互**

我们将传统的音频播放器彻底重构，引入了 Apple 风格的“灵动岛”设计。

- **极客交互**：默认呈现紧凑的 CD 旋转态，悬停时丝滑拉伸为磨砂玻璃长岛，露出歌名与控制键。
- **弹幕歌词 (Danmaku)**：首创单轨道屏幕弹幕歌词，让文字像流星一样划过屏幕，带给阅读一种沉浸式的陪伴感。
- **双模态无缝衔接**：
    - **站点模式**：左下角常驻，负责全站背景随机音乐。
    - **文章模式**：自动识别文内音频块，点击即刻接管全局引擎，开启专属封面与歌词同步。
- **拟物化按键**：为播放、切歌键打造了细腻的“玻璃下沉”压感反馈，手感极佳。

## **🤠 新增可选开关（按需启用）**

为了让首页更符合不同人的审美与信息密度偏好，我为 HeoPro 增加了两个“可选开关”，你可以在主题配置中自由控制首页组件的显示与隐藏。

- **英雄区开关**：用于控制首页顶部英雄区（大卡区域）是否显示。
    - 配置项：`HEO_HERO_ENABLE`
    - 位置：`themes/heo/config.js`
    - 取值：`true` 显示 / `false` 隐藏
- **首页分类栏开关**：用于控制首页英雄区下方的文章分类栏（`CategoryBar`）是否显示。
    - 配置项：`HEO_HOME_CATEGORY_BAR_ENABLE`
    - 位置：`themes/heo/config.js`
    - 取值：`true` 显示 / `false` 隐藏
